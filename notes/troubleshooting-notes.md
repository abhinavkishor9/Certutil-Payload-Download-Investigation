# 🔧 Troubleshooting Notes

## Overview

This document summarizes the issues encountered during the **Certutil Payload Download Investigation** and the corresponding observations and resolutions.

---

## Issue 1: Local HTTP Server Not Accessible

### Problem

The browser was unable to connect to the local web server.

**Error:**

```
This site can't be reached
localhost refused to connect
```

### Cause

The local HTTP server was not running or was configured incorrectly.

### Resolution

Start the local HTTP server and verify its availability by browsing to:

```
http://localhost:8000/test.txt
```

If the browser displays:

```
Hello from Certutil Lab
```

the server is functioning correctly.

---

## Issue 2: Certutil Returned "Access is denied"

### Command

```cmd
certutil -urlcache -split -f http://localhost:8000/test.txt test.txt
```

### Output

```
Access is denied.
```

### Cause

Modern versions of Windows may restrict access to the WinINet cache used by the `-urlcache` option. As a result, the download operation may fail even though the command executes successfully.

### Resolution

No further action was required for this investigation because the primary objective was to analyze the **Sysmon Event ID 1** generated by the execution of `certutil.exe`.

---

## Issue 3: File Download Failed

### Observation

The test file was not downloaded successfully.

### Analysis

Although the download failed, **Sysmon Event ID 1 (Process Creation)** was successfully generated and contained:

- Process Image
- Parent Process
- Command Line
- User Context
- Process ID
- Process GUID
- Timestamp

This information is sufficient for SOC investigation and threat hunting.

---

## Lessons Learned

- Successful payload delivery is **not required** for detecting suspicious LOLBin activity.
- Failed execution attempts can still provide valuable forensic evidence.
- Sysmon Event ID 1 captures detailed process creation telemetry useful for detection and incident response.
- Monitoring the command-line execution of trusted Windows binaries such as `certutil.exe` is an effective detection strategy.

---

## Conclusion

Although the payload retrieval did not succeed due to local Windows restrictions, the attempted execution of **certutil.exe** generated sufficient telemetry for investigation. From a SOC perspective, attempted LOLBin abuse should be treated as suspicious activity and analyzed further.
